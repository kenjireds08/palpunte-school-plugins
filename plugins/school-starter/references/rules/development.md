# 開発ルール

## 開発原則
- 要件定義を最優先
- 既存パターンを踏襲
- テスト実行後にデプロイ
- 複雑なタスクは3-5段階に分割
- エラー発生時は最大3回まで試行
- 依頼された内容のみ実装し、過剰な機能追加・抽象化をしない
- 推測でコードを提案せず、必ず該当ファイルを確認してから回答する

## コード実装前の必須確認
- 類似の既存コードを確認
- コードスタイル・命名規則を踏襲
- 既存のユーティリティを活用

## セキュリティ
- .envファイルはコミットしない
- APIキーはVercel環境変数で管理
- 認証情報をログに出力しない

## テストコマンド
```
npm run lint       # lint
npm run typecheck  # 型チェック
npm test           # テスト
npm run build      # ビルド
npm run check      # 一括実行（あれば）
```

## 検証ルール
実装完了後は検証コマンドを実行し、エラーがなくなるまで修正すること
