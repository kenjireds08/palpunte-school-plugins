# エラー解決パターン集

案件で遭遇したエラーと解決策を自動蓄積するファイル。
全プロジェクトのClaudeが参照・追記する。

## 記録ルール
- エラーを2回以上試行して解決した場合に自動追記する（確認不要）
- 同じエラーパターンが既にあれば追記せず、解決策が異なる場合のみ追加する
- エラーに遭遇したら先にこのファイルを確認する

## 記録フォーマット

### [カテゴリ] エラーの簡潔な説明
- **発生日**: YYYY-MM-DD
- **案件**: プロジェクト名
- **エラー**: エラーメッセージ（核心部分のみ）
- **失敗したアプローチ**: 試して駄目だった方法
- **解決策**: 最終的に成功した方法
- **根本原因**: なぜこのエラーが起きたか（1行）

## カテゴリ一覧
- [Next.js] - ビルド、ルーティング、SSR/RSC関連
- [Supabase] - RLS、Auth、Edge Functions、マイグレーション
- [TypeScript] - 型エラー、設定、互換性
- [Tailwind/CSS] - スタイリング、レスポンシブ
- [Git/Deploy] - Vercel、GitHub Actions、デプロイ
- [API連携] - 外部API、認証、レート制限
- [npm/依存関係] - パッケージ競合、バージョン不整合
- [Claude Code] - MCP、Skills、Hooks、設定
- [その他] - 上記に該当しないもの
