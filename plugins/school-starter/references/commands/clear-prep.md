# Clear準備コマンド

コンテキストをクリアする前に、**詳細な**引き継ぎ用のメッセージをターミナルに出力します。

## 重要なルール

- **ファイルやドキュメントを作成しない**
- **ターミナルに直接テキストを出力する**
- ユーザーがコピペして /clear 後に貼り付けられる形式にする
- **次のClaudeが困らないように、詳細に記載する**

## 出力フォーマット

以下の形式でターミナルに直接出力してください。**各セクションは省略せず、詳細に記載すること**：

```
========================================
📋 引き継ぎメッセージ
========================================

## 🎯 プロジェクト
- プロジェクト名: [名前]
- パス: [絶対パス]
- 概要: [何をするプロジェクトか1-2行で]

## ✅ 今回やった作業（詳細）
### 1. [作業カテゴリ1]
- 何をしたか具体的に
- どのファイルを変更したか
- なぜそうしたか（意思決定の理由）

### 2. [作業カテゴリ2]
- 同様に詳細を記載

## 📁 作成・更新したファイル一覧
### 新規作成
- `[絶対パス]` - [ファイルの役割・内容の説明]
- `[絶対パス]` - [ファイルの役割・内容の説明]

### 更新
- `[絶対パス]` - [何を変更したか]
- `[絶対パス]` - [何を変更したか]

### 削除
- `[絶対パス]` - [削除した理由]

## ⚙️ 技術的な決定事項
- [決定1]: [採用した理由]
- [決定2]: [採用した理由]

## 🚫 試したが却下した選択肢
- [選択肢1]: [却下した理由]
- [選択肢2]: [却下した理由]

## 💡 重要な発見・学び
- [発見1]
- [発見2]

## 🔄 次にやるべき作業
### 最優先
1. [タスク] - [具体的に何をすべきか]

### その後
2. [タスク] - [具体的に何をすべきか]
3. [タスク] - [具体的に何をすべきか]

## ⚠️ 注意事項・ブロッカー
- [あれば詳細に記載、なければ「特になし」]
- [既知の問題や制約]

## 📍 現在の状態
- コードの状態: [ビルド成功/失敗、テスト状態]
- 開発サーバー: [起動中/停止中]
- 未コミットの変更: [あり/なし]
- 最後のコミット: [コミットメッセージ]

## 📚 参照すべきドキュメント
- `[パス]` - [いつ参照すべきか]
- `[パス]` - [いつ参照すべきか]

========================================
↑ これをコピーして /clear 後に貼り付けてください
========================================
```

## 詳細記載のガイドライン

### なぜ詳細が必要か
- /clear 後のClaudeは**何も覚えていない**
- 省略された情報は**失われる**
- 詳細であればあるほど、スムーズに作業を再開できる

### 特に詳しく書くべきこと
1. **ファイルパス**: 必ず絶対パスで記載
2. **意思決定の理由**: なぜその選択をしたか
3. **試行錯誤の過程**: 何を試して何がダメだったか
4. **現在の状態**: ビルド・テスト・サーバーの状態
5. **次のアクション**: 具体的に何をすべきか

### 省略してはいけないこと
- 作成したファイルの一覧
- 変更したファイルの一覧
- 技術的な決定事項
- エラーや問題が発生した場合はその詳細

## 手順

1. 今回の会話内容を**全て**振り返る
2. 上記フォーマットで**詳細な**引き継ぎメッセージを作成
3. ターミナルに直接出力する（ファイル作成禁止）
4. ユーザーに「コピーして /clear 後に貼り付けてください」と伝える
