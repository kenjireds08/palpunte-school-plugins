# コードレビュー（2段階方式）

## 目的

実装が終わったコードを、**2段階**でレビューする。

1. **内部レビュー** — `feature-dev:code-reviewer` サブエージェント（Claude Code 内の別スレッド）で静的レビュー。critical/high は修正ループで自動対応。
2. **独立レビュー準備** — 最後に「作業サマリー（Codex独立レビュー用）」を必ず出力。受講生はこれを **Codex 別タブ（CLI または ChatGPT）に貼り付けて第三者レビュー**を受ける。

この2段階により、Claude Code 内レビュー（同一モデルの盲点が残る可能性あり）と Codex レビュー（別モデル＝独立視点）の両方を得る。サンドボックスと干渉しないコピペ方式なので、セキュリティ設定と両立する。

## いつ発動するか

以下のような発話で自動発動する:

- 「レビューして」「コードレビューして」
- 「仕上げにチェックして」「最終チェックして」
- 「レビューお願い」「approveしてもらって」
- コミット前・PR作成前・納品前のタイミング

## 前提

`feature-dev` プラグインがインストール済みであること。未導入の場合は以下を案内:

```
feature-dev プラグインが必要です。以下を実行してください:
  /plugin install feature-dev@claude-plugins-official
インストール後、Claude Code を再起動してからもう一度レビューを依頼してください。
```

## フロー

### Step 1: レビュー対象の特定

```bash
git diff HEAD --stat
git diff HEAD --name-status --find-renames
```

未コミットの変更を対象とする。変更がなければ「レビュー対象がありません」と報告して終了。

### Step 2: feature-dev:code-reviewer でレビュー

`Agent` ツールで `subagent_type: "feature-dev:code-reviewer"` を呼び出す。プロンプト例:

```
直近の未コミット変更をレビューしてください。
観点: セキュリティ / ロジックバグ / パフォーマンス / プロジェクト規約違反 / 見落とし
高信頼度の指摘のみに絞り、ノイズを避けてください。
各指摘には severity (critical/high/medium/low) を明示してください。
```

### Step 3: 修正ループ（最大5回 / approve まで繰り返す）

`critical` / `high` の指摘があれば、approve になるまで以下を繰り返す:

1. Claude Code 本体が**最小差分**で修正（仕様変更は未解決issue扱いで別タスク化）
2. テスト / lint が用意されていれば実行
3. `feature-dev:code-reviewer` をもう一度呼び出して再レビュー

**停止条件（いずれかに該当したら Step 4 へ）:**
- critical/high が 0 件（= approve）
- 5回到達 → 未解決事項を明記して Step 4 へ
- 2回連続で同じ指摘が残る（修正が効いていない）→ ユーザーへ相談
- 修正で別の critical/high が新たに増えた → 一旦停止してユーザーへ相談
- テスト2回連続失敗 → 原因特定を優先し、ユーザーへ相談

### Step 4: 最終サマリー出力（絶対省略禁止）

**この出力は必ず行う。** 変更が小さくても、ファイルが1つでも、critical が0件でも、必ず出力する。
受講生はこのサマリーを**コピペして別タブの Codex で独立レビュー**を受ける運用。省略すると受講生のワークフローが止まる。

以下の形式で出力:

~~~markdown
---

## 作業サマリー（Codex独立レビュー用）

### 完了タスク
- タスク1: 概要
- タスク2: 概要

### 変更ファイル
| ファイル | 変更内容 |
|---------|---------|
| path/to/file.tsx | 変更の説明 |

### 主な実装ポイント
- ポイント1（設計判断・採用した方針）
- ポイント2

### feature-dev:code-reviewer 指摘と対応
- [high] ファイル:行 — 指摘内容 → 対応: 修正済み / 未解決（理由）
- [medium] ファイル:行 — 指摘内容 → 対応: 修正 / 保留（advisory）

### ビルド状態
- TypeScript: ✅ / ❌
- ESLint: ✅ / ⚠️ / ❌ / 未設定
- Build: ✅ / ❌ / 未実行

### 未解決・懸念点（あれば）
- config.ts L55: 内容、リスク、推奨アクション

### Codexへの依頼文（そのままコピペしてください）

> 以下の差分を独立視点でレビューしてください。
> 観点: セキュリティ / ロジック抜け / パフォーマンス / Claude が見落としがちなエッジケース
>
> 作業サマリー:
> （上記の「完了タスク」〜「未解決・懸念点」をそのまま貼る）
>
> 差分:
> ```diff
> （`git diff HEAD` の主要部分を貼り付け。長い場合は主要ファイルに絞る）
> ```

---
~~~

### Step 5: 受講生への案内

サマリー出力後、以下を明示する:

```
✅ Claude Code 側のレビュー完了（feature-dev:code-reviewer）。
📋 次のステップ: 上記「作業サマリー」をコピーして、別タブの Codex または ChatGPT に貼り付けて独立レビューを受けてください。

【Codex 別タブで実行する場合】
  別ターミナルで `codex` を起動 → サマリーを貼り付け → 「レビューして」

【ChatGPT で実行する場合】
  ChatGPT を開く → 新しいチャット → サマリーを貼り付け → 「レビューして」

Codex から追加指摘があれば、もう一度「レビューして」と言ってください。
この2段階を回すことで、Claude Code の盲点を Codex がカバーします。
```

## エラー時の対応

| 状況 | 対応 |
|------|------|
| `feature-dev` 未インストール | 上記「前提」のインストール案内を表示して終了 |
| `feature-dev:code-reviewer` が失敗 | 原因を報告し、手動レビューまたは再試行を案内 |
| git diff で変更なし | 「レビュー対象がありません」と報告して終了（Step 4は実行しない） |
| 修正ループで進展しない | 5回で打ち切り、未解決事項を Step 4 に明記して出力 |

## 設計方針

- **サブエージェント経由**: サンドボックス内でも安全に動作する（外部バイナリ呼び出しなし）
- **コピペ方式**: Codex との統合はコピペに限定することで、sandbox/Codex CLI の競合を回避
- **サマリー必須化**: 最終サマリーの出力を絶対に省略しないことで、受講生が第三者レビューのステップを飛ばさない仕組みに
- **明示的な依頼文**: Codex へ渡す依頼文を完成形で用意し、受講生が迷わずコピペできるようにする
